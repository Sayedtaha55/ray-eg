# 1) نظرة عامة على المشروع

## 1.1 تعريف مختصر
**Ray** منصة Marketplace متعددة الأدوار، تجمع التجارة الإلكترونية مع إدارة تشغيلية للتاجر والتوصيل والإدارة العامة.

## 1.2 المشكلة التي يحلّها المشروع
المشروع يستهدف فجوة شائعة في السوق المحلي:
- متاجر كثيرة لا تملك نظام بيع/إدارة رقمي متكامل.
- العميل يحتاج تجربة موحدة (تصفح، طلب، حجز، عروض).
- الإدارة تحتاج أدوات رقابة وتحليل وتشغيل.

## 1.3 القيمة الأساسية
- تمكين التاجر من البيع وإدارة نشاطه من لوحة واحدة.
- تسهيل شراء/حجز العميل من واجهات عامة سريعة.
- توفير دور واضح للكابتن في دورة الطلب والتوصيل.
- تمكين الإدارة من الحوكمة والتحليلات والمتابعة.

## 1.4 الأدوار (Roles)

### Customer
- يتصفح المتاجر/المنتجات/العروض.
- يطلب أو يحجز.
- يدير ملفه الشخصي وتتبع حالته.

### Merchant
- يدير بيانات المتجر.
- يدير المنتجات والمخزون والعروض.
- يتابع الطلبات والمؤشرات التشغيلية.

### Courier
- يغيّر حالته (متاح/غير متاح).
- يستقبل ويقبل/يرفض عروض التوصيل.
- يتابع الطلبات المخصصة له.

### Admin
- يدير الموافقات والحوكمة.
- يتابع الـ feedback والتنبيهات.
- يشرف على الحالة العامة للنظام.

## 1.5 نطاق المزايا الرئيسي
- Authentication (بما فيها Google OAuth)
- Shops / Products
- Orders / Reservations / Invoices
- Offers / Analytics
- Notifications / Feedback
- Media Uploads / Image Maps
- Courier workflows

## 1.6 لمحة تقنية
- Frontend: React + TypeScript + Vite + Tailwind
- Backend: NestJS (Modular Monolith style)
- ORM/DB: Prisma + PostgreSQL (أساسي) مع وجود SQLite legacy paths
- Runtime إضافي: Redis (حسب الحالة)

## 1.7 حدود المشروع الحالية (As-Is)
- الكود يجمع واجهات متعددة داخل نفس مشروع Frontend.
- الـ Backend يدعم تشغيل جزئي للموديولات عبر env.
- توجد مسارات legacy/schema متعددة يجب التعامل معها بحذر.
